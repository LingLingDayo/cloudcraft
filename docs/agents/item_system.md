# 道具与物品系统架构规范 (Item & Block System)

本规范明确了道具与物品系统的设计原则与实现要求。在添加新物品、定义方块配置或扩展交互行为时，必须严格遵守以下规范，以确保各模块高内聚、低耦合，并避免循环依赖。

---

## 1. 核心概念与继承体系

物品与方块是两个解耦的系统。方块（Block）描述世界中网格体素的物理与渲染属性，而物品（Item）描述玩家背包与工具栏中的实体道具。

### 核心类关系
* **Item**：所有物品的基类，定义了基本的物品属性（如标识、名称、堆叠大小）和通用的多态交互接口。
* **BlockItem** (`category: 'block'`)：方块物品，其内部持有对应的方块标识，负责具体方块的物理碰撞检查与放置操作。
* **FoodItem** (`category: 'food'`)：食物物品，携带营养/生命回复参数，并处理食用逻辑。

---

## 2. 依赖解耦与 Properties Resolver 模式

方块配置定义与方块实例注册表采用 **Properties Resolver（属性解析器）运行时注入模式** 进行解耦：

1. **结构定义与属性接口**：方块配置模块仅负责定义属性结构接口（如 `BlockProperties`）与提供公共查询入口 `getBlockProperties(id)`，不直接静态引用任何具体方块配置实例。
2. **注册与注入**：所有方块的具体配置在 `BlockRegistry` 静态初始化时统一注册。加载完成后，将查询函数注入到配置模块中。
3. **获取属性**：其他模块获取方块属性时，统一调用运行时属性查询接口，该接口通过被注入的解析器向 `BlockRegistry` 发起查询，从而打破了静态依赖闭环。

## 3. 右键交互与多态分发

交互行为由手持物品类的方法多态分发，而非硬编码在外部逻辑中进行大量的条件分支判断。

### 交互上下文（解耦与依赖注入）
为了防止 `Item` 及其子类直接依赖引擎大类，采用上下文对象传递交互所需数据，避免直接持有复杂引用：
* **ItemUseContext**：当右键空气直接使用物品（如食用食物、饮用药水）时传入。它封装了当前玩家的状态属性（如饱食度、生命值）和游戏模式，用于决策物品能否使用。
* **BlockPlaceContext**：当右键对准世界中的方块使用物品（如放置方块、种植作物、使用工具）时传入。它封装了当前的世界实例、瞄准位置、待放置位置、朝向面、玩家物理碰撞箱等物理与世界环境数据，用于处理放置检测和物理阻挡。
* **ItemUseResult**：物品使用后返回的轻量副作用描述对象（如是否消耗、恢复数值的增量等），由交互管理模块统一接收并应用副作用，实现逻辑与副作用应用的分离。

> [!IMPORTANT]
> 上下文的属性可能会随着物理引擎或游戏业务的发展而不断扩展。编写具体交互逻辑时，请**直接查阅类型定义**中上下文的最新属性，禁止在文档中硬编码具体属性期望。

### 交互决策链流转
当触发右键交互时，交互链按以下顺序执行：
1. **方块交互**：如果对准了方块，且该方块是可交互的（例如箱子、拉杆等），调用 `Block` 自身的交互方法。若处理成功，终止流程。
2. **对方块使用物品 (onUseOnBlock)**：如果手持物品，调用 `item.onUseOnBlock(BlockPlaceContext)`。若返回 `true`，代表物品已成功在此方块上执行了动作（如放置或种植），终止流程。
3. **直接使用物品 (onUse)**：如果上述步骤未拦截，调用 `item.onUse(ItemUseContext)`。如果返回了有效的 `ItemUseResult`，则由交互管理模块统一应用副作用（如扣减背包、播放音效、更新玩家属性状态等）。

## 4. 最佳实践指南 (如何添加新内容)

### A. 添加一个新方块 (并自动成为可放置物品)
1. 在类型声明模块的方块类型枚举/常量中定义新方块的 ID。
2. 在 `BlockRegistry` 初始化时：
   * 实例化对应的方块类（或通用方块类）并调用注册接口。
   * **注意**：方块注册后，物品注册表会根据该方块的配置自动包装生成同名的 `BlockItem`，无需手动在物品注册表中重复声明。

### B. 添加一个纯道具物品（如食物、工具等）
1. 在类型声明模块的物品类型枚举/常量中定义道具的 ID。
2. 在 `ItemRegistry` 初始化时：
   * 注册对应类型的实例。如果是食物，实例化并注册 `FoodItem` 并指定其回复参数（如生命值和饱食度增量）。
3. 在本地化配置文件中配置对应的翻译文案。

### C. 添加新的右键交互效果
* 如果想给某个物品增加独特的右键对空使用效果：
  * 重写其 `onUse(ctx: ItemUseContext): ItemUseResult | null`，决策是否可使用并返回对应的副作用描述。
* 如果想给物品增加独特的对准方块效果（例如锄地、修剪）：
  * 重写其 `onUseOnBlock(ctx: BlockPlaceContext): boolean` 并执行对应逻辑，执行成功后返回 `true`。
